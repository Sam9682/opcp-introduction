LACP, Trunk & RAID

Intermediate

Overview

This lesson explains three infrastructure concepts that customers may encounter when discussing their OPCP environment. Each concept is presented with a plain-language definition, a real-world analogy, and guidance on when a customer would benefit from it.

These are behind-the-scenes technologies that make cloud services faster, more organised, and more resilient. You do not need to understand how they work technically — only what they achieve for the customer.

LACP — Link Aggregation

What It Is

LACP (Link Aggregation Control Protocol) is a method of combining multiple network cables into a single, faster, and more reliable connection. Instead of relying on one cable to carry all network traffic, several cables work together as a team.

In one sentence: LACP combines multiple network connections into one faster, more reliable link — like adding extra lanes to a motorway.

Real-World Analogy

Multiple lanes on a motorway: A single-lane road can only handle a limited number of cars at once. If that lane is blocked, all traffic stops. A multi-lane motorway carries more traffic simultaneously, and if one lane is closed for maintenance, the others keep flowing. LACP does the same thing for network connections — more capacity and built-in backup if one connection fails.

Customer Benefits

  • Increased speed — More data can flow at the same time, reducing bottlenecks during peak usage
  • Built-in resilience — If one cable fails, the others continue working without interruption
  • No single point of failure — The connection remains active even during partial hardware issues

When a Customer Needs This

A customer benefits from LACP when their workload requires high network throughput (large file transfers, streaming, many simultaneous users) or when network reliability is critical to their business operations (e-commerce, real-time services, financial transactions).

Trunk — Network Traffic Organisation

What It Is

A network trunk is a method of organising different types of network traffic into separate channels over a single physical connection. It allows multiple distinct networks to share the same cable without interfering with each other.

In one sentence: Trunking organises network traffic into separate channels on one connection — like sorting mail into different mailboxes at the same address.

Real-World Analogy

Sorting mail into different mailboxes: Imagine an apartment building with one front door but individual mailboxes for each resident. All mail arrives through the same entrance, but it is sorted so each resident only receives their own letters. A network trunk works the same way — all traffic travels over one connection, but it is tagged and sorted so each network only sees its own data.

Customer Benefits

  • Clean separation — Different departments, projects, or security zones can share infrastructure without seeing each other's traffic
  • Simplified cabling — One physical connection carries multiple logical networks, reducing complexity
  • Security boundaries — Sensitive traffic (like payment processing) stays isolated from general traffic (like email)
  • Efficient resource use — No need for separate physical cables for each network segment

When a Customer Needs This

A customer benefits from trunking when they need to keep different types of traffic separated for security or organisational reasons — for example, isolating production systems from development environments, separating guest Wi-Fi from internal networks, or meeting compliance requirements that mandate data segregation.

Software RAID — Data Protection

What It Is

RAID (Redundant Array of Independent Disks) is a method of protecting data by storing copies across multiple storage drives. If one drive fails, the data is still safe on the remaining drives, and the system continues operating without interruption.

In one sentence: RAID protects data by storing copies across multiple drives — like keeping backup copies of important documents in separate filing cabinets.

Real-World Analogy

Backup copies in separate filing cabinets: Imagine you have an important contract. Rather than keeping only one copy in one cabinet (where a fire or flood could destroy it), you keep identical copies in three different cabinets in different rooms. If one cabinet is damaged, you still have the complete document in the others. Software RAID does this automatically with digital data — copies are maintained across multiple drives so that no single drive failure causes data loss.

Customer Benefits

  • Data safety — Information survives hardware failures without any loss
  • Continuous operation — The system keeps running even when a drive fails; the faulty drive can be replaced without downtime
  • Peace of mind — Critical business data is protected against the most common type of hardware failure
  • Automatic recovery — When a failed drive is replaced, the system rebuilds the missing copy automatically

When a Customer Needs This

A customer benefits from RAID when they store data that cannot be easily recreated or where loss would have significant business consequences — customer databases, financial records, intellectual property, or any system where data integrity is non-negotiable.

Quick Comparison

Concept What It Does Analogy Key Benefit
LACP Combines multiple network cables into one faster connection Multiple lanes on a motorway Faster, more reliable network
Trunk Organises different traffic types into separate channels Sorting mail into different mailboxes Clean separation and security
RAID Stores data copies across multiple drives Backup copies in separate filing cabinets Data survives hardware failures

What to Tell Customers

  • These technologies work behind the scenes to make their service faster, more organised, and more resilient
  • LACP ensures their network connection is both fast and fault-tolerant — no single cable failure causes an outage
  • Trunking keeps their different workloads securely separated without needing separate physical infrastructure
  • RAID means their data is protected against the most common type of hardware failure — a single drive breaking
  • All three are managed by the platform — customers benefit from them without needing to configure anything themselves
