Choosing the Right Node

Not every job requires the same tool. You wouldn't use a delivery truck to commute to work, and you wouldn't try to move furniture in a compact car. The same principle applies to nodes in OPCP — different workloads need different types of computing power.

Think of node types like choosing a vehicle. Each type is designed for specific tasks, and picking the right one means better performance and better value for your customer.

Node Types Comparison

Standard Nodes — The Compact Car

Just like a compact car is perfect for daily commuting and errands, Standard nodes are ideal for everyday computing tasks that don't require exceptional power.

Best Suited For

  • Web applications — Company websites, internal portals, and lightweight web services
  • Development and testing — Building and testing software before it goes live
  • Small databases — Customer lists, product catalogs, and internal records
  • Email and collaboration tools — Supporting team communication

Customer Conversation Tip

Recommend Standard nodes when a customer says: "We need to host our company website" or "We want a development environment for our team."

Performance Nodes — The SUV

An SUV handles rougher terrain and carries more passengers. Performance nodes deliver more computing power for applications that need to handle heavier loads or respond faster.

Best Suited For

  • Business-critical applications — ERP systems, CRM platforms, and financial software
  • Medium-scale databases — Transaction processing, order management
  • Multi-user platforms — Applications serving hundreds of simultaneous users
  • Containerized workloads — Running multiple services on a single node

Customer Conversation Tip

Recommend Performance nodes when a customer says: "Our application needs to handle peak traffic during business hours" or "We're running our core business software."

High-Memory Nodes — The Truck

A truck is built to carry heavy loads that other vehicles simply can't handle. High-Memory nodes provide large amounts of memory for workloads that need to process or hold massive amounts of data at once.

Best Suited For

  • Large data processing — Analyzing millions of records, generating complex reports
  • In-memory databases — Applications that keep entire datasets in memory for speed
  • Analytics platforms — Business intelligence tools processing large data sets
  • High-traffic applications — Services handling thousands of simultaneous connections

Customer Conversation Tip

Recommend High-Memory nodes when a customer says: "We need to run real-time analytics on our entire customer database" or "Our application processes large files in memory."

How to Help Customers Choose

When discussing node types with customers, focus on what they need to accomplish rather than technical specifications. Ask questions like:

  • "How many people will use this application at the same time?"
  • "How much data does your application work with?"
  • "Is this for development/testing or for production use?"
  • "Does your application need to respond very quickly, or is some delay acceptable?"

Their answers will naturally point toward the right node type. And remember — customers can always start with a Standard node and upgrade later as their needs grow.

Key Takeaways

  • Standard nodes are for everyday workloads — like a compact car for daily commuting
  • Performance nodes handle demanding applications — like an SUV for tougher terrain
  • High-Memory nodes process large data — like a truck for heavy loads
  • Focus on customer needs, not technical specs, when recommending a type
  • Customers can upgrade node types as their business grows
