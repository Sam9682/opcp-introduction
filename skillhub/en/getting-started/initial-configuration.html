Initial Configuration

Beginner

Overview

After logging in for the first time, there are a few setup steps to complete before your OPCP environment is ready to use. Think of this like moving into a new office: the building is ready, but you still need to set up your desk, connect the phone, and get your security badge configured.

This lesson walks you through the essential first-time configuration steps using plain, non-technical language.

Step 1: Select Your Datacentre Location

Your first decision is choosing where your private cloud resources will physically live. A datacentre is a secure facility that houses the computers running your services.

Choose a location based on:

  • Proximity to your users — Closer locations mean faster response times
  • Legal requirements — Some regulations require data to stay within certain countries
  • Redundancy needs — You may want resources in multiple locations for backup purposes

Step 2: Set Up Your Private Network

Next, you create a vRack — your private network. This is the secure connection between all your cloud services. Imagine it as a private hallway connecting different rooms in your office: only your team can walk through it.

Setting up a vRack involves:

  1. Giving your private network a name you will recognise (e.g., "Production Network")
  2. Choosing which services should be connected to it
  3. Confirming the setup

Step 3: Configure Your First Host

A host is the physical server that provides computing power for your private cloud. Configuring your first host means choosing how powerful it should be — similar to choosing between a compact car and a lorry depending on what you need to carry.

During host configuration, you will select:

  • Host type — Different sizes for different workloads (small for testing, large for production)
  • Quantity — How many hosts you need to start with (you can add more later)
  • Datacentre assignment — Which location this host belongs to

Step 4: Add Storage

Your host needs somewhere to store data. Adding a datastore is like adding a filing cabinet to your new office — it gives your applications a place to save their work.

Choose your storage based on:

  • Capacity — How much space you need (measured in gigabytes or terabytes)
  • Performance tier — Standard storage for everyday use, or high-performance storage for demanding applications

Step 5: Set Up User Access

Finally, decide who on your team should have access to the OPCP environment and what they should be allowed to do. This is like issuing security badges — each person gets access only to the areas they need.

You can create user accounts with different permission levels:

  • Administrator — Full access to all settings and services (for IT managers)
  • Operator — Can manage services but cannot change billing or account settings
  • Read-only — Can view information but cannot make changes (useful for auditors or observers)

Configuration Complete

Once these five steps are done, your OPCP environment is ready for use. Your team can begin deploying applications and storing data in your private cloud. Here is a quick recap:

  1. Datacentre location chosen
  2. Private network (vRack) created
  3. First host configured
  4. Storage added
  5. Team access set up
