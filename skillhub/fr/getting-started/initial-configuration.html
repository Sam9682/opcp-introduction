Configuration Initiale

Débutant

Présentation

Après votre première connexion, quelques étapes de configuration sont nécessaires avant que votre environnement OPCP soit prêt à l'emploi. C'est comme emménager dans un nouveau bureau : le bâtiment est prêt, mais vous devez encore installer votre poste de travail, brancher le téléphone et faire configurer votre badge de sécurité.

Cette leçon vous guide à travers les étapes essentielles de configuration initiale en utilisant un langage simple et non technique.

Étape 1 : Sélectionner l'emplacement de votre centre de données

Votre première décision est de choisir où vos ressources de cloud privé seront physiquement hébergées. Un centre de données est une installation sécurisée qui héberge les ordinateurs exécutant vos services.

Choisissez un emplacement en fonction de :

  • La proximité avec vos utilisateurs — Des emplacements plus proches signifient des temps de réponse plus rapides
  • Les exigences légales — Certaines réglementations imposent que les données restent dans certains pays
  • Les besoins de redondance — Vous pouvez souhaiter des ressources dans plusieurs emplacements à des fins de sauvegarde

Étape 2 : Configurer votre réseau privé

Ensuite, vous créez un vRack — votre réseau privé. C'est la connexion sécurisée entre tous vos services cloud. Imaginez-le comme un couloir privé reliant différentes pièces de votre bureau : seule votre équipe peut l'emprunter.

La mise en place d'un vRack implique :

  1. Donner à votre réseau privé un nom que vous reconnaîtrez (ex. : « Réseau Production »)
  2. Choisir quels services doivent y être connectés
  3. Confirmer la configuration

Étape 3 : Configurer votre premier hôte

Un hôte est le serveur physique qui fournit la puissance de calcul pour votre cloud privé. Configurer votre premier hôte signifie choisir sa puissance — un peu comme choisir entre une voiture compacte et un camion selon ce que vous devez transporter.

Lors de la configuration de l'hôte, vous sélectionnerez :

  • Type d'hôte — Différentes tailles pour différentes charges de travail (petit pour les tests, grand pour la production)
  • Quantité — Combien d'hôtes vous avez besoin pour commencer (vous pourrez en ajouter plus tard)
  • Affectation au centre de données — À quel emplacement cet hôte appartient

Étape 4 : Ajouter du stockage

Votre hôte a besoin d'un endroit pour stocker les données. Ajouter un datastore, c'est comme ajouter une armoire de classement à votre nouveau bureau — cela donne à vos applications un endroit pour sauvegarder leur travail.

Choisissez votre stockage en fonction de :

  • Capacité — L'espace dont vous avez besoin (mesuré en gigaoctets ou téraoctets)
  • Niveau de performance — Stockage standard pour un usage quotidien, ou stockage haute performance pour les applications exigeantes

Étape 5 : Configurer les accès utilisateurs

Enfin, décidez qui dans votre équipe doit avoir accès à l'environnement OPCP et ce qu'ils doivent être autorisés à faire. C'est comme distribuer des badges de sécurité — chaque personne n'a accès qu'aux zones dont elle a besoin.

Vous pouvez créer des comptes utilisateurs avec différents niveaux de permissions :

  • Administrateur — Accès complet à tous les paramètres et services (pour les responsables informatiques)
  • Opérateur — Peut gérer les services mais ne peut pas modifier la facturation ou les paramètres du compte
  • Lecture seule — Peut consulter les informations mais ne peut pas effectuer de modifications (utile pour les auditeurs ou observateurs)

Configuration terminée

Une fois ces cinq étapes accomplies, votre environnement OPCP est prêt à l'emploi. Votre équipe peut commencer à déployer des applications et stocker des données dans votre cloud privé. Voici un récapitulatif rapide :

  1. Emplacement du centre de données choisi
  2. Réseau privé (vRack) créé
  3. Premier hôte configuré
  4. Stockage ajouté
  5. Accès de l'équipe configuré

À partir d'ici, vous pouvez explorer la section Concepts Fondamentaux pour en savoir plus sur le fonctionnement de vos ressources, ou passer aux Opérations Techniques pour une vue d'ensemble de la gestion quotidienne.
