Understanding Network Architecture

In OPCP, nodes don't operate in isolation — they need to communicate with each other and with the outside world. Network architecture defines how these connections are organized, secured, and optimized.

Think of it like a city's road system. There are main highways (public network) that connect the city to the rest of the world, and internal streets (private network) that link buildings to each other. Some roads are wider and faster than others, and security barriers control who can access what.

OPCP Network Topology

Public Network — The Main Highway

The City Analogy: Highways

The public network is like the highway that connects your city to the rest of the world. This is where visitors (users, customers) arrive to access your services.

What This Means in OPCP

The public network allows your nodes to be accessible from the internet:

  • End users access your applications via the public network
  • Each node can receive a public IP address (like a visible street address)
  • Traffic is protected by firewalls and security rules
  • Bandwidth determines how many visitors can access simultaneously

For customers: The public network is what makes their applications accessible to their own clients and users.

Private Network — The Internal Streets

The City Analogy: Residential Streets

The private network is like the internal streets of a residential neighborhood. Only residents (your nodes) can travel on them. Outside visitors don't have direct access.

What This Means in OPCP

The private network allows your nodes to communicate with each other securely:

  • Nodes exchange data without going through the internet (faster and more secure)
  • The database communicates with the web application without external exposure
  • Internal traffic is isolated from public traffic (like a gated community)
  • No bandwidth charges for internal communications

For customers: The private network ensures that sensitive data (like database information) never travels over the public internet.

Bandwidth — The Width of the Road

The City Analogy: Number of Lanes

Bandwidth is like the number of lanes on a road. A two-lane road can handle a certain volume of traffic, but during rush hour, it becomes congested. A six-lane highway handles many more vehicles simultaneously.

What This Means for Customers

  • More bandwidth = more users can access services at the same time
  • Less bandwidth = sufficient for low-traffic applications
  • E-commerce sites during sales events need more bandwidth
  • An internal tool used by 10 people needs little bandwidth

Network Security — Barriers and Controls

The City Analogy: Checkpoints

Just as a city has checkpoints at entrances and surveillance cameras, the OPCP network has security mechanisms to protect your nodes:

  • Firewalls — Like guards at the entrance who check who can enter
  • Filtering rules — Like "no entry" signs on certain roads
  • Network isolation — Like walls between neighboring properties
  • Encryption — Like sending messages in sealed envelopes

Key Takeaways

  • The public network connects your nodes to the internet — it's the main highway to the outside world
  • The private network lets nodes communicate with each other securely — these are the internal streets
  • Bandwidth determines how much traffic can flow — it's the width of the road
  • Network security protects your nodes with firewalls and filtering rules
  • The public/private separation is essential for protecting sensitive data
