Configuration d'Instance

Intermédiaire

Présentation

Configurer une nouvelle instance dans OPCP est similaire à l'emménagement dans un nouveau bureau. Vous choisissez la taille et l'agencement dont vous avez besoin, confirmez les détails, puis l'espace est préparé pour vous. Cette page vous guide à travers le processus complet à un niveau conceptuel — aucune commande technique n'est requise.

Analogie : Pensez à la configuration d'instance comme à la commande d'un ordinateur sur mesure dans un catalogue. Vous sélectionnez les spécifications dont vous avez besoin, confirmez votre commande, et le système est assemblé et livré prêt à l'emploi.

Le Processus de Configuration

La création d'une nouvelle instance suit un processus clair, étape par étape, via l'interface de gestion. Voici ce que chaque étape implique :

Étape 1 : Choisir un Modèle

Commencez par sélectionner un modèle préconfiguré qui correspond à vos besoins. Les modèles sont comme des plans d'appartement — chacun propose une combinaison recommandée de puissance de traitement, de mémoire et de stockage adaptée à un type de travail particulier.

  • Usage Général — Ressources équilibrées pour les applications métier courantes
  • Optimisé Calcul — Puissance de traitement supplémentaire pour les tâches intensives en données
  • Optimisé Mémoire — Mémoire additionnelle pour les applications qui traitent de grands volumes de données simultanément
  • Optimisé Stockage — Capacité de stockage étendue pour l'archivage ou les charges de travail riches en médias

Étape 2 : Configurer les Ressources

Après avoir sélectionné un modèle, vous ajustez l'allocation des ressources. C'est comme réorganiser les meubles après avoir choisi votre appartement — vous décidez exactement de la quantité de chaque ressource dont vous avez besoin.

  • Capacité de traitement — La puissance de calcul dédiée à votre instance
  • Allocation mémoire — La mémoire de travail disponible pour les tâches actives
  • Volume de stockage — L'espace disque attribué à vos données
  • Connectivité réseau — Les réseaux avec lesquels votre instance peut communiquer

Étape 3 : Nommage et Organisation

Donnez un nom significatif à votre instance et assignez-la à un projet ou un département. C'est comme étiqueter les pièces d'un bâtiment — cela aide tout le monde à trouver et gérer les ressources efficacement.

  • Choisissez un nom descriptif qui identifie l'objectif (ex. : « Analytique Marketing » ou « Portail Client »)
  • Assignez au groupe organisationnel approprié pour la facturation et le contrôle d'accès
  • Ajoutez des étiquettes optionnelles pour faciliter la recherche et le filtrage ultérieurs

Étape 4 : Vérification et Confirmation

Avant la création de l'instance, vous voyez un récapitulatif de tous vos choix — modèle, ressources, nommage et coûts estimés. C'est votre occasion de tout vérifier, comme relire une commande avant de cliquer sur « Acheter ».

  • Vérifiez le modèle sélectionné et les niveaux de ressources
  • Confirmez le nommage et l'assignation organisationnelle
  • Consultez le coût mensuel estimé
  • Approuvez la configuration pour lancer le provisionnement

Étape 5 : Déploiement

Une fois confirmée, la plateforme provisionne automatiquement votre instance. Cela prend généralement quelques minutes — similaire à l'attente qu'un nouveau téléphone termine sa configuration initiale. Vous recevrez une notification lorsque l'instance sera prête à l'emploi.

  • Le système alloue les ressources demandées à partir de la capacité disponible
  • Les connexions réseau sont établies automatiquement
  • L'instance apparaît dans votre tableau de bord avec le statut « Prêt »
  • Vous pouvez commencer à l'utiliser immédiatement ou la transmettre à votre équipe technique

Ce qu'il Faut Dire aux Clients

  • La configuration d'instance est un processus guidé et visuel — aucune expertise technique n'est requise pour l'initier
  • Les modèles simplifient la prise de décision en proposant des configurations pré-validées
  • L'ensemble du processus, de la sélection à une instance opérationnelle, prend généralement moins de 10 minutes
  • Les coûts sont transparents et visibles avant tout engagement
  • Les instances peuvent être modifiées après leur création si les besoins évoluent
