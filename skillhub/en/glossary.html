Glossary

🇫🇷 Français

This glossary defines the key technical terms used throughout the OPCP Skillhub. Each term is explained in plain language with a real-world comparison to help you understand and remember the concept.

A

API (Application Programming Interface)
A set of rules that allows different software systems to communicate with each other. Think of it like a waiter in a restaurant — you tell the waiter what you want (your request), the waiter takes it to the kitchen (the system), and brings back your food (the response). You never need to enter the kitchen yourself.

B

Bandwidth
The maximum amount of data that can be transferred over a network connection in a given time. Like the number of lanes on a motorway — more lanes mean more cars (data) can travel simultaneously without congestion.
Backup
A copy of data stored separately from the original, used for recovery if the original is lost or damaged. Like photocopying important documents and keeping the copies in a different location.

C

CloudStore
OVHcloud's storage solution that provides scalable, secure space for your data in the cloud. Think of it as a secure, expandable warehouse where you can store as many boxes (files) as you need, accessible from anywhere.
Configuration
The process of setting up a node or system with specific settings for its intended purpose. Like furnishing and arranging a new office to suit how your team works.
CPU (Central Processing Unit)
The brain of a computer that performs calculations and processes tasks. Like the number of employees in a department — more CPUs mean more tasks can be handled simultaneously.
Credentials
The username, password, or security keys used to prove your identity when accessing a system. Like a key card that grants you access to specific floors in an office building.

D

Datacentre
A secure, climate-controlled building filled with powerful computers (servers) that store and process data. OVHcloud operates datacentres across Europe, North America, and Asia-Pacific.
Datastore
A storage volume attached to your private cloud where files, databases, and application data are kept. Think of it as a hard drive for your cloud servers.
Decommissioning
The orderly process of shutting down and removing a node when it is no longer needed. Like moving out of an office — you pack your belongings, clean up, and return the keys.

H

Host
A physical server within your private cloud that provides computing power (processing and memory). Think of it as a powerful workstation dedicated entirely to your organisation's tasks.

L

LACP (Link Aggregation Control Protocol)
A method of combining multiple network cables into a single, faster, and more reliable connection. Like adding extra lanes to a motorway — more capacity and built-in backup if one lane is closed.
Least Privilege
A security principle where each user is given only the minimum access they need to do their job — nothing more. Like giving office keys only to the rooms each person actually needs to enter.
Load Balancer
A system that distributes incoming work across multiple servers to prevent any single server from being overwhelmed. Like a store greeter directing customers to the shortest checkout queue.

N

Network Architecture
The design and organisation of how computers and systems are connected to each other. Like a city's road system — highways connect to the outside world, internal streets link buildings together.
Node
A compute resource unit within the OPCP infrastructure — essentially a powerful virtual computer dedicated to running your workloads. Like an individual office in a managed building.

O

OPCP (OVHcloud Hosted Private Cloud)
A dedicated cloud infrastructure service where your organisation gets its own private computing resources, separate from other customers. Like having a private apartment in a professionally managed building — your own space with shared building services.
Operation
The active phase of a node's lifecycle where it is running workloads and delivering value. Like the daily use of an office — people come in, do their work, and the space serves its purpose.

P

Private Network
An internal network that connects your nodes to each other without going through the public internet. Like internal corridors in an office building — only people inside the building can use them.
Provisioning
The process of requesting and setting up a new node with the required resources. Like signing a lease for a new office — you choose the size and location, and the space is prepared for you.
Public Network
The network connection that allows your nodes to be accessible from the internet. Like the main entrance to a building — it's how visitors (users) reach your services from the outside world.

R

RAID (Redundant Array of Independent Disks)
A method of protecting data by storing copies across multiple storage drives. If one drive fails, the data is still safe on the others. Like keeping backup copies of important documents in separate filing cabinets.
RAM (Random Access Memory)
Short-term working memory that a computer uses to handle active tasks. Like the size of your desk — a bigger desk lets you spread out more documents and work on multiple things without constantly going to the filing cabinet.
Resource Allocation
The process of dividing available computing resources (CPU, RAM, storage) among your workloads. Like dividing a department budget — you distribute funds wisely across different needs.

S

Storage
Permanent space where data is kept, even when the system is turned off. Like a filing cabinet — the bigger it is, the more documents you can keep long-term.

T

Trunk (Network Trunking)
A method of organising different types of network traffic into separate channels over a single physical connection. Like sorting mail into different mailboxes at the same address — all mail arrives through one entrance but each resident only receives their own.

V

vRack
OVHcloud's private network technology that connects your cloud services to each other securely. Think of it as a private corridor between offices that only your team can use.
