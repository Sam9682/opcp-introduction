📖 Glossary Beginner

Quick Reference

One-page summary cards for each section of the OPCP training. Use these as a handy reminder during customer conversations.

Introduction

Main Topics

  • What OPCP is and how it works (dedicated cloud infrastructure managed by OVHcloud)
  • Business benefits: cost predictability, security, scalability, sovereignty
  • Target audiences: mid-to-large enterprises, regulated industries, growing businesses
  • Key features: dedicated resources, VMware-based virtualisation, network isolation, API access

Key Terminology

  • OPCP — OVHcloud Hosted Private Cloud: a dedicated cloud environment hosted and managed by OVHcloud
  • Hosted Private Cloud — Infrastructure reserved exclusively for one customer, not shared with others
  • Hypervisor — Software that creates and runs virtual machines on physical hardware

Customer Talking Points

  • "Your own dedicated infrastructure without the cost of building a data centre."
  • "European data sovereignty with ISO 27001 and SOC 2 certifications."
  • "Scale resources up or down without long-term hardware commitments."

Getting Started

Main Topics

  • Account creation and initial setup process
  • Accessing the OPCP management dashboard
  • Navigating the control panel interface
  • Initial configuration steps for a new environment

Key Terminology

  • Control Panel — The web interface where customers manage their OVHcloud services
  • Dashboard — The main overview screen showing resource status at a glance
  • NIC Handle — The unique customer identifier used to log in to OVHcloud services

Customer Talking Points

  • "You can be up and running within hours, not weeks."
  • "The dashboard gives you a single view of all your resources and their status."
  • "No command-line expertise needed — everything is manageable through the web interface."

Core Concepts

Main Topics

  • Node lifecycle: provisioning → configuration → operation → decommissioning
  • Node types and their workload suitability (compute-optimised, memory-optimised, storage-optimised)
  • Resource allocation: how CPU, RAM, and storage are assigned
  • Network architecture: public vs. private networks, bandwidth, isolation

Key Terminology

  • Node — A physical server dedicated to your Private Cloud environment
  • vCPU — Virtual CPU: a share of processing power allocated to a virtual machine
  • VLAN — Virtual Local Area Network: an isolated network segment for security and organisation
  • Datastore — A storage volume attached to your Private Cloud for virtual machine disks

Customer Talking Points

  • "Choose the node type that matches your workload — no paying for power you do not need."
  • "Resources can be added or removed without downtime."
  • "Private networks keep your internal traffic completely isolated from the public internet."

Technical Operations

Main Topics

  • Instance setup workflow (request → provision → configure → validate)
  • API credentials: what they enable, how they are managed
  • Node configuration options and their business impact
  • Advanced networking: LACP (link aggregation), Trunk (VLAN tagging), Software RAID (disk redundancy)

Key Terminology

  • API — Application Programming Interface: allows software to interact with OPCP programmatically
  • LACP — Link Aggregation: combining multiple network connections for speed and reliability
  • Trunk — A single connection carrying traffic for multiple VLANs
  • RAID — Redundant Array of Independent Disks: protects data by spreading it across multiple drives

Customer Talking Points

  • "Automation via API means repetitive tasks can be handled without manual intervention."
  • "Link aggregation provides both faster speeds and automatic failover if a connection drops."
  • "RAID ensures that a single disk failure does not result in data loss."

Storage Solutions

Main Topics

  • CloudStore overview: object storage for unstructured data
  • Storage capabilities: scalability, redundancy, access methods, capacity tiers
  • Data management: organisation, lifecycle policies, retention
  • Backup and recovery: automated backups, restore procedures, disaster recovery

Key Terminology

  • CloudStore — OVHcloud's scalable object storage service
  • Object Storage — A method of storing data as discrete objects (files) rather than in a traditional file system hierarchy
  • Redundancy — Storing multiple copies of data across different locations to prevent loss
  • RPO/RTO — Recovery Point Objective (how much data you can afford to lose) / Recovery Time Objective (how quickly you need to be back online)

Customer Talking Points

  • "Storage scales automatically — you only pay for what you use."
  • "Built-in redundancy means your data is protected even if hardware fails."
  • "Automated backups run on schedule so you never forget to protect critical data."

Best Practices

Main Topics

  • Operations: resource monitoring, scaling strategies, maintenance schedules
  • Security: access control (least privilege), encryption, compliance (ISO 27001, SOC 2)
  • Performance: right-sizing, caching, load balancing
  • Troubleshooting: common issues (slow response, connectivity, storage), escalation paths

Key Terminology

  • Least Privilege — Giving users only the minimum access they need to do their job
  • Right-Sizing — Matching resource allocation to actual workload requirements
  • Load Balancer — A system that distributes incoming traffic across multiple servers
  • SLA — Service Level Agreement: the guaranteed uptime and performance commitments

Customer Talking Points

  • "Regular monitoring prevents surprises — you will see issues before your users do."
  • "OVHcloud's certifications mean your compliance requirements are already covered at the infrastructure level."
  • "Multiple support channels ensure you can always get help when you need it."
