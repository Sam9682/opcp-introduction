API Usage & Credential Management

Intermediate

Overview

APIs (Application Programming Interfaces) allow different software systems to communicate with each other automatically. In the context of OPCP, APIs enable your organisation's tools and workflows to interact with the cloud platform without manual intervention.

Real-world analogy: Think of an API as a universal remote control. Instead of walking up to each device and pressing buttons manually, you use the remote to send instructions from a distance. Credentials are like the batteries and pairing code — they give the remote permission to control specific devices.

What APIs Enable

APIs open up powerful capabilities for organisations that want to streamline their cloud operations:

Automation

Rather than performing repetitive tasks manually through the dashboard, APIs allow your technical team to set up automated workflows. For example:

  • Automatically creating new instances when demand increases
  • Scheduling regular backups without human intervention
  • Scaling resources up during business hours and down overnight
  • Generating usage reports on a set schedule

Customer benefit: Automation reduces human error and frees your team to focus on higher-value work instead of routine maintenance.

Integration with Existing Tools

APIs allow OPCP to connect with the tools your organisation already uses:

  • Monitoring dashboards that track resource usage across all your systems
  • Billing systems that consolidate cloud costs with other expenses
  • Ticketing systems that automatically provision resources when requests are approved
  • Security tools that enforce compliance policies across your infrastructure

Customer benefit: Integration means OPCP fits into your existing workflows rather than requiring you to adopt entirely new processes.

Self-Service Capabilities

With API access, authorised team members can perform actions programmatically that would otherwise require support tickets or manual dashboard work:

  • Provisioning development environments on demand
  • Adjusting resource allocations based on real-time needs
  • Retrieving detailed usage analytics for planning purposes

Customer benefit: Faster response times and reduced dependency on support channels for routine operations.

How Credentials Are Obtained

Credentials are the secure keys that grant API access. They are managed through the OPCP manager interface — the same web-based dashboard used for other administrative tasks.

The Credential Request Process

Obtaining API credentials follows a controlled, auditable process:

  1. Request access — An authorised administrator navigates to the credentials section of the manager interface
  2. Define scope — The administrator specifies what the credentials should be allowed to do (read-only, full management, specific services only)
  3. Generate credentials — The system creates a unique set of credentials tied to the defined scope
  4. Secure delivery — Credentials are displayed once for the administrator to record securely; they cannot be retrieved again after this point
  5. Activate — The credentials become active and can be used immediately

Analogy: This is like issuing a building access card. You decide which doors it opens (scope), create the card (generate), hand it to the employee (delivery), and it works immediately (activate).

Types of Credentials

  • Full-access credentials — Can perform any operation; typically reserved for senior administrators
  • Scoped credentials — Limited to specific actions or services; recommended for most use cases
  • Read-only credentials — Can retrieve information but cannot make changes; ideal for monitoring and reporting tools

Credential Rotation

Credential rotation means periodically replacing existing credentials with new ones. This is a security best practice, similar to changing the locks on your office doors on a regular schedule.

Why Rotation Matters

  • Limits exposure — If credentials are accidentally shared or compromised, rotation ensures they stop working after a defined period
  • Compliance requirements — Many industry regulations require regular credential changes
  • Audit trail — Each rotation creates a clear record of when access was refreshed

The Rotation Process

  1. Generate new credentials — Create a fresh set through the manager interface
  2. Update systems — Replace the old credentials in all tools and workflows that use them
  3. Verify functionality — Confirm that automated processes continue working with the new credentials
  4. Revoke old credentials — Deactivate the previous set so they can no longer be used

Analogy: Rotation is like changing your email password regularly. You create a new password, update it on all your devices, verify everything still works, and the old password stops functioning.

Recommended Rotation Frequency

The appropriate rotation schedule depends on the sensitivity of what the credentials access:

  • High-sensitivity (full-access credentials) — Every 30 to 90 days
  • Standard (scoped credentials) — Every 90 to 180 days
  • Low-sensitivity (read-only credentials) — Every 180 to 365 days

What to Tell Customers

  • APIs allow their existing tools to work directly with OPCP, reducing manual effort
  • Credentials are managed through a secure, auditable process in the manager interface
  • Access can be scoped to follow the principle of least privilege — each credential only has the permissions it needs
  • Regular rotation is straightforward and helps meet compliance requirements
  • No programming knowledge is needed to generate or manage credentials — it is all done through the visual interface
